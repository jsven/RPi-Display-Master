
**角色设定：**  
你是一位资深的全栈开发工程师，精通 Next.js、Tailwind CSS 以及 Linux/树莓派系统底层配置。你擅长编写高效、稳定的 Bash 脚本，并能将其转化为易于使用的 Web 交互产品。

**项目背景：**  
我们需要开发一个名为 **“RPi-Display Master”** 的在线工具。目的是帮助购买了非标准尺寸 HDMI 屏幕（如长条屏、圆屏、工业屏）的树莓派用户，通过网页选择配置，一键生成适配脚本，解决屏幕黑屏、分辨率异常或触摸错位的问题。

**技术要求：**

1. **前端框架：** Next.js (App Router), Tailwind CSS, Lucide-React (图标)。
2. **后端逻辑：** Next.js API Routes (用于处理参数并拼接生成 Bash 脚本)。
3. **部署目标：** 适配 linux,windows 或普通轻量级服务器。

**功能模块要求：**

1. **核心配置数据库 (JSON 格式)：**
    - 定义一个 `screens.json` 库， 参考 https://github.com/goodtft/LCD-show 包含至少 3 个示例：
        * 型号 A: "8.8寸长条屏 (480x1920)"，含特定 `hdmi_cvt` 参数。
        * 型号 B: "7寸标准屏 (1024x600)"。
        * 型号 C: "5寸圆屏 (1080x1080)"。
    - 每个配置需包含：分辨率、HDMI 时序参数、触摸驱动类型（如 I2C/USB）。
2. **用户交互界面 (UI)：**
    - **第一步：** 选择屏幕型号（带缩略图图标的卡片式选择）。
    - **第二步：** 选择系统版本（树莓派 OS Bookworm 64位、Bullseye 32位等）。
    - **第三步：** 选择旋转方向（0°, 90°, 180°, 270°），点击时有视觉示意。
    - **生成按钮：** 点击后通过动画展示正在生成。
3. **脚本生成逻辑 (Backend Logic) 脚本需要参考 https://github.com/Botspot/pi-apps ：**
    - 脚本需要包含：自动备份原 `config.txt`、检查并写入 `hdmi_group/mode/cvt`、根据旋转角度计算触摸转换矩阵（Calibration Matrix）。
    - 脚本应具备容错性，例如检测是否为 root 权限运行。
    - 脚本需要兼容在不同的环境运行 参考 https://github.com/openvinotoolkit
4. **结果展示区域：**
    - 展示一键执行指令，例如：`curl -sL https://yourdomain.com/api/install?id=88bar&rot=90 | sudo bash`。
    - 提供“复制”按钮。
    - 下方列出该配置的具体技术参数摘要，增加专业感。

**任务输出：**  
请为我生成以下文件的完整代码：

1. `src/data/screens.json`: 屏幕参数配置文件。
2. `src/app/page.tsx`: 主交互界面代码（响应式设计）。
3. `src/app/api/generate/route.ts`: 后端脚本组装引擎逻辑。
4. `public/scripts/core_engine.sh`: 包含通用逻辑（备份、写入、重启）的 Bash 模板。
